# Module 3: CLI workflows for newsrooms

## End of the week message

You finished the toughest technical week of the course.

This week you built a working pipeline: fetch article, extract content, summarize with AI, format for output. That's the pattern behind most automation in newsrooms. Once you can chain these steps, you can adapt them to almost any repetitive task.

**What you learned:**
- How to describe a workflow in plain English and have the AI build it
- How to use `/plan` to review an approach before anything gets built
- How to test on small batches before running at scale
- The error debugging loop — paste the error, get the fix
- API key security basics

**Exercise reminder:** Your pipeline submission is due Sunday at 11:59pm. Post a screenshot of your working script, one newsletter item it generated, and one idea for adapting the pipeline to your newsroom.

**Next week:** Module 4 covers agents, `claude -p` (non-interactive mode for scripting), and connecting AI to external data sources. You'll learn how AI systems that can take actions and access knowledge bases are changing newsroom workflows — including real implementations like The Washington Post's Haystacker and The New York Times's Echo. You'll also configure Claude Code to query a local knowledge base using MCP, and you'll use `claude -p` to run prompts as part of automated scripts without an interactive session.

If you struggled this week, that's expected. Building automation scripts and debugging failures are unfamiliar territory for most journalists. The important thing is you saw the pattern: describe the workflow, review the plan, test small, fix what breaks. Everything else is iteration.

The skills from this week feed directly into the final project, where you'll build a tool for your own newsroom.
