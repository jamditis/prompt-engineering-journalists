Why leave the browser?

The sandbox problem

  • Web AI tools run inside a sandbox — isolated from your computer
  • Can't leave a scraper running overnight
  • Can't process a dataset that won't fit in a text box
  • Can't write output directly to your project folder

CLI tools change this. They give the AI access to your actual machine: filesystem, network, and compute power.

What is a CLI?

A text-based way to interact with your computer.

  • Instead of clicking buttons, you type commands
  • Think of it as keyboard shortcuts for your entire computer
  • Every Mac and PC has a terminal app built in
terminal

$ claude

You type claude and press Enter. That's it.

The trade-off

Web advantages
  • No setup required
  • Visual formatting (markdown, tables)
  • Easy to share conversations
  • Works on any device with a browser
CLI advantages
  • Direct file access — no uploading
  • Real compute power (not capped by browser)
  • Automation and scripting
  • Pipe output between tools
  • Works offline (for local models)

Side-by-side demo

Same task, both tools, same model

Claude.ai (browser)
  • Faster response time
  • Surface-level report
  • Citation links in response
  • Hit tool-use limit mid-task
  • Output lives only in chat window
Claude Code (CLI)
  • Slower — Claude thinks through each step
  • Saved files directly to disk
  • Deeper extraction from sources
  • Materials persist after session ends
  • Output ready to use immediately

Both used Opus 4.6. The model was the same. The harness was different.
