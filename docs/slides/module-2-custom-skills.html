The problem

You type the same instructions over and over.

You explain how to extract action items from meeting minutes. Then again the next day. Then to a colleague. Different phrasing each time, different results each time.

Skills break that cycle. Write it once, use it forever.

Level 2

Saved prompt

You write the perfect prompt once and save it in a document. Copy-paste when you need it.

  • Consistent wording every time
  • Still requires manual copy-paste
  • You have to remember it exists
  • Lives outside your tool — Claude doesn't know about it
STATUS: Better, but you're still the middleware.
Module 2: Custom skills 6
Level 3

Custom slash command

Save the prompt as a markdown file in .claude/commands/. The filename becomes the command.

FILE
.claude/commands/source-check.md
INVOKE
/source-check
  • Claude loads it automatically when you type the slash
  • Versioned in Git, shareable with your whole newsroom
  • Same result every time, no copy-paste
Module 2: Custom skills 7
Level 4

Hook

Runs in the background. You don't invoke it — it fires when certain conditions are met.

Automated second reader

Like having someone check your work on every file, every time, without being asked.

  • No manual invocation needed
  • Enforces standards you'd otherwise forget
  • The highest level of automation in this stack
Module 2: Custom skills 8
02
Skills vs commands

Two kinds of saved knowledge

Skill

Contains domain knowledge. Claude applies it with judgment.

Like a style guide — reference material, contextual interpretation.

  • Flexible, context-aware
  • Claude decides how to apply it
Command

Deterministic trigger. Same steps, same sequence, every time.

Like a pre-publication checklist — fixed order, no skips.

  • Rigid, predictable
  • Same output given same input

Notify

Notify hooks in practice

Automated second reader

Like a copy editor marking up your draft while you keep writing.

  • Flags unattributed quotes
  • Catches AI filler language and slop patterns
  • Checks for style guide violations
  • Work continues — you review the flags when you're ready

KEY IDEA: Non-blocking. Flags issues without stopping your work.

Stop hooks in practice

One-way door guardian

"This is about to do X — proceed?"

  • Pauses before deleting files
  • Requires approval before pushing code
  • Blocks publishing without human review

EDITORIAL RULE: Does a human need to see this before it goes out?
