Module 3

Custom skills for Claude Code

Extend Claude Code with reusable skills that encode domain expertise, workflows, and best practices for journalism.

Learning objectives

  • Understand what Claude Code skills are and how they work
  • Install skills from public repositories and personal collections
  • Use skills to apply specialized knowledge to journalism tasks
  • Evaluate the quality and trustworthiness of third-party skills

Key concepts

What are skills?

Skills are markdown files containing instructions, prompts, and context that Claude Code loads on demand. When you invoke a skill (e.g., /source-verification), Claude reads the skill file and applies that knowledge to your conversation. Skills let you encode expertise once and reuse it across projects.

Skill types

Claude Code supports three types of skill files:

  • SKILL.md: Domain expertise and methodology. Use when working on tasks that match the skill's domain.
  • RULE.md: Project constraints and requirements. Defines what must and must not be done.
  • LESSON.md: Learned patterns from past projects. Captures what worked, what failed, and why.

Where skills live

Skills can be stored in several locations:

  • Global: ~/.claude/commands/ - Available in all projects
  • Project: .claude/commands/ - Available only in that project
  • Remote: GitHub repositories that you clone or reference

Skill trust and safety

Skills are just text files with instructions - they can't run arbitrary code. However, they can influence Claude's behavior, so review skills before installing them. Prefer skills from trusted sources, check what instructions they contain, and be cautious of skills that ask Claude to access external services or sensitive data.

Exercise: Install and use the source-verification skill

Goal: Install a journalism-focused skill and use it to verify a claim from a viral social media post.

Step 1: Create your commands directory

If you haven't already, create the global commands directory:

mkdir -p ~/.claude/commands

Step 2: Create the source-verification skill

Create a new file at ~/.claude/commands/source-verification.md with the following content:

# Source verification skill

You are helping a journalist verify claims and sources. Follow these principles:

## Verification checklist
1. **Identify the claim**: What specific factual assertion is being made?
2. **Find the original source**: Trace the claim back to its origin. Screenshots and social posts are not primary sources.
3. **Check multiple sources**: Look for at least 2-3 independent sources confirming the claim.
4. **Evaluate source credibility**: Consider the source's track record, expertise, and potential biases.
5. **Look for counter-evidence**: Actively search for information that contradicts the claim.
6. **Check the date**: Is this current information or old news being recirculated?
7. **Verify images/video**: Consider whether media could be manipulated, out of context, or from a different event.

## Red flags
- Claim originated on social media without attribution
- No primary source documents available
- Single source with strong bias
- Emotional language designed to provoke outrage
- Missing or vague dates and locations
- "Trust me" or "I heard from someone who..."

## Response format
When asked to verify a claim, provide:
1. The specific claim being evaluated
2. Sources found (with URLs when available)
3. Verification status: VERIFIED / UNVERIFIED / FALSE / PARTIALLY TRUE / UNABLE TO VERIFY
4. Confidence level: HIGH / MEDIUM / LOW
5. Key evidence supporting your conclusion
6. Any caveats or limitations in your verification

Always recommend additional steps the journalist should take for complete verification.

Step 3: Verify the skill is available

Start Claude Code and check that the skill appears:

claude
/help

You should see source-verification listed among available commands.

Step 4: Test the skill with a viral claim

Let's verify a hypothetical viral claim. In Claude Code, invoke the skill:

/source-verification

Then provide a claim to verify:

A viral tweet claims that a major US city just banned all plastic bags, effective immediately, with $500 fines for violations. The tweet has 50,000 retweets. Please help me verify this claim before I report on it.

Step 5: Review Claude's response

Notice how Claude structures its response according to the skill's format. It should:

  • Identify the specific factual claims
  • Note the lack of primary source
  • Suggest verification steps (check city council records, contact city communications office)
  • Flag red flags (viral spread, no attribution)

Step 6: Try a real verification

Now try verifying a real claim you've encountered. Use the skill to structure your fact-checking process.

Troubleshooting

Skill doesn't appear in /help

Make sure the file is in the correct directory (~/.claude/commands/) and has the .md extension. The filename becomes the command name.

Skill doesn't seem to affect Claude's responses

Remember to invoke the skill with the slash command (e.g., /source-verification) before asking your question. The skill context is loaded when you invoke it.

"Permission denied" when creating files

Make sure the ~/.claude/commands/ directory exists and you have write permission. Run mkdir -p ~/.claude/commands to create it.

Skill is too generic or not helpful

Skills work best when they're specific. Edit the skill file to add more detailed instructions, examples, or checklists relevant to your work.
