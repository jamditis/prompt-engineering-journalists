MAC · TROUBLESHOOTING

Fix PATH after installing Claude Code

You installed Claude Code, you typed claude, and your terminal said command not found. This is almost always a PATH problem. Here's what that means and the three-step fix that works for most Macs.

THE SYMPTOM

You installed Claude Code (or npm said "installed successfully"), you typed claude, and your terminal responded with:

zsh: command not found: claude

What PATH actually is

When you type a command in the terminal, your shell has to figure out which program to run. It doesn't search your whole hard drive. It looks only in a specific list of folders called PATH.

PATH is just a list of folders separated by colons, stored in an environment variable. When you type claude, the shell walks through each folder in PATH, in order, and runs the first claude file it finds. If Claude Code installed itself into a folder that isn't in PATH, the shell can't find it, and you get "command not found" even though the file is definitely on your computer.

This is the single most common Mac issue with CLI tools. It's not a bug in Claude Code — it's how shells on Unix systems have worked since the 1970s.

Step 1: Restart your terminal

Most installers update your shell config file, but that config only gets read when a new shell starts. If you installed Claude Code in a terminal window that was already open, that window won't know about the updates yet.

Completely quit Terminal (Cmd+Q) and reopen it. Don't just close the window — quit the app. Then try again:

claude

If this works, you're done. If it still says "command not found," continue to Step 2.

Step 2: Find out where Claude Code actually is

Before you fix PATH, you need to know where the installer put the claude binary. In a fresh terminal, try:

npm list -g --depth=0 2>/dev/null | head
npm config get prefix

The first command lists anything you installed globally with npm. The second prints the folder npm uses as its install prefix — usually something like /usr/local, /opt/homebrew, or ~/.npm-global. The claude binary should be in the bin subfolder of whatever that prefix is.

If those two commands don't point you at it, check the usual install locations directly — this is much faster than searching the whole disk:

command -v claude || find /usr/local/bin /opt/homebrew/bin "$HOME/.npm-global/bin" -name claude -type f 2>/dev/null

If that prints a full path ending in /claude, write down the directory that contains it (usually a bin folder), not the full path. PATH entries have to be directories. You'll need that folder for Step 3.

Step 3: Add that folder to your PATH

macOS uses zsh by default (has since Catalina, released in 2019). Your zsh config lives in ~/.zshrc. If that file doesn't exist yet, create it — it's just a plain text file.

Open it in any text editor:

open -e ~/.zshrc

Add this line at the bottom, replacing /path/to/folder with the directory you identified in Step 2 (not a full path ending in /claude — PATH entries must be directories):

export PATH="$PATH:/path/to/folder"

For example, if your npm prefix is ~/.npm-global, the line would be:

export PATH="$PATH:$HOME/.npm-global/bin"

Save the file and close TextEdit. Then reload the config in your current terminal:

source ~/.zshrc

Now try claude again. It should launch.

If you're using bash instead of zsh

Some Macs — especially older ones or ones set up before 2019 — still default to bash. To check which shell you're in, run:

echo $SHELL

If it says /bin/bash, use ~/.bash_profile (or ~/.bashrc) instead of ~/.zshrc in Step 3. The rest of the fix is identical.

The escape hatch: reinstall with the native installer

If the PATH surgery in Step 3 feels like too much, you can start over using the official native installer from Anthropic, which handles PATH automatically. Download the installer first so you can eyeball what it does before running it — this is good hygiene for any curl | sh command you see on the internet:

curl -fsSL https://claude.ai/install.sh -o install.sh
less install.sh
sh install.sh

The first line downloads the script from Anthropic's official domain. less lets you read through it (press q to quit). The third line runs it, dropping the claude binary into a folder that's already in your PATH on most Macs. After it finishes, quit Terminal, reopen it, and try claude.

If you had previously installed via npm install -g @anthropic-ai/claude-code, the two installs can coexist but both provide a claude command, and which one runs depends on PATH order. If things still behave strangely after reinstalling, run which claude to see which binary your shell picks up, and remove or disable the npm install with npm uninstall -g @anthropic-ai/claude-code if you want the native one to win.

IF NONE OF THIS WORKS

Post in the Technical Questions for the instructor forum with three things: (1) the output of echo $SHELL, (2) the output of echo $PATH, and (3) the output of which claude. Those three together are usually enough for Joe or a classmate to diagnose it remotely.
