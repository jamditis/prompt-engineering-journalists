Skills
A skill is a markdown file that becomes a slash command
  • Lives in .claude/commands/
  • Filename = command name: source-check.md → /source-check
  • Plain English instructions — no programming language
  • Only loads when invoked — keeps context window clean
  • Versioned in Git, shareable with colleagues
Think of it as a colleague you pull into a conversation for their specific expertise, then let go back to their desk.
--- allowed-tools: Bash, Read, Write, Grep --- # Social media audit Audit a public official's social media presence across platforms. ## Steps 1. Find accounts on all platforms 2. Download public video content 3. Transcribe audio 4. Screenshot every second 5. Detect on-screen text 6. Run content analysis 7. Build interactive results page

Plugins
A plugin is a package of skills + hooks, installed from GitHub
  • Bundle of skills and hooks someone else has built
  • Install with one command from a GitHub repo
  • All skills show up as slash commands instantly
  • All hooks start running automatically
  • Can include agents, settings, and MCP server configs
EXAMPLE
$ claude install jamditis/claude-skills-journalism Installed: 36 skills, 13 hooks
Skills included

Source verification, FOIA requests, meeting minutes extraction, data cleaning, content formatting, beat-specific tools

Hooks included

AI filler language detection, one-way-door checks before publishing, unattributed quote flags, style guide enforcement

Comparison
Skills vs. plugins
Skill

A single tool you build yourself

What it is One markdown file
Where it lives .claude/commands/
How you get it Write it yourself
How you use it Type /command-name
Scope One task
Analogy A recipe card you wrote
Plugin

A toolkit someone packaged and shared

What it is A GitHub repo with skills + hooks + config
Where it lives Installed globally by Claude Code
How you get it claude install user/repo
How you use it Skills appear as commands; hooks run automatically
Scope An entire workflow or domain
Analogy A cookbook someone published
You start by writing skills. When you have enough, you can package them as a plugin and share on GitHub.
