Module 2

Voice-controlled AI

Use speech-to-text to interact with AI tools hands-free, enabling new workflows for interviews, dictation, and accessibility.

Learning objectives

  • Understand available voice transcription options and their trade-offs
  • Configure voice input for Claude Code using different backends
  • Distinguish between RAW mode and Agent mode for voice workflows
  • Build a practical voice-to-text workflow for story outlining

Key concepts

Why voice input matters

Voice input lets you brainstorm without context-switching to your keyboard, dictate notes while reviewing documents, and work hands-free when typing isn't practical. For journalists, this means capturing thoughts during research, outlining stories while commuting, or making AI tools accessible for those with mobility limitations.

Transcription backends

Several options exist for converting speech to text:

  • Gemini Flash: Google's fast, accurate API. Requires a Google Cloud account. Good accuracy, low latency.
  • OpenAI Whisper: Can run locally or via API. Excellent accuracy, especially for technical terms. Local version is free but slower.
  • Windows Speech Recognition: Built into Windows. No setup required, works offline, but less accurate than cloud options.
  • macOS Dictation: Built into macOS. Press the microphone key twice or use Fn Fn. Works offline with recent macOS versions.

RAW mode vs. Agent mode

RAW mode: Your speech is transcribed and sent directly to Claude without modification. Good for dictating content verbatim or when you want full control over the prompt.

Agent mode: Claude Code interprets your speech as instructions and may read files, run commands, or take multi-step actions. Good for hands-free coding or document processing.

Latency and accuracy trade-offs

Cloud transcription (Gemini, Whisper API) offers better accuracy but adds network latency. Local options (Whisper local, OS built-in) work offline but may be slower or less accurate. For real-time conversation, optimize for latency. For document dictation, optimize for accuracy.

Exercise: Set up voice input and dictate a story outline

Goal: Configure voice input for Claude Code and use it to dictate a story outline for a hypothetical news article.

Step 1: Choose your transcription backend

For this exercise, we'll use your operating system's built-in dictation as the simplest option. You can later upgrade to Whisper or Gemini for better accuracy.

On macOS:

  • Go to System Settings > Keyboard > Dictation
  • Turn on Dictation
  • Note the shortcut (default: press Control twice, or Fn twice on newer Macs)

On Windows:

  • Press Win + H to enable voice typing
  • Or go to Settings > Time & Language > Speech to configure

Step 2: Test basic dictation

Open any text editor and activate dictation using your shortcut. Speak a few sentences to confirm it's working:

"The quick brown fox jumps over the lazy dog. Testing one two three."

Check that punctuation is handled correctly. Most systems recognize "period," "comma," "question mark" as punctuation commands.

Step 3: Open Claude Code in your terminal

Navigate to a project folder and start Claude Code:

cd ~/Documents/stories
claude

Step 4: Dictate your prompt

With Claude Code waiting for input, activate dictation and speak your prompt. Try this scenario:

"I'm working on a story about a local school board meeting where they voted to change the start time for high schools from 7:30 AM to 8:30 AM. Help me outline the story. I need to cover the vote itself, the reasons behind it, parent reactions, and what happens next."

Review the transcribed text before pressing Enter. Make corrections if needed.

Step 5: Iterate using voice

After Claude responds with an outline, continue the conversation using voice:

"Add a section about potential impacts on after-school activities and bus schedules."
"What sources should I contact for this story?"

Step 6: Save your outline

Ask Claude to save the completed outline:

"Save this outline to a file called school-time-outline.md"

Troubleshooting

Dictation isn't activating

Make sure dictation is enabled in your system settings. On macOS, you may need to download the language model first. On Windows, check that the Speech service is running.

Transcription is inaccurate

Speak clearly and at a moderate pace. Reduce background noise. If accuracy is critical, consider using Whisper or a cloud API instead of built-in dictation.

The wrong words are being transcribed

Technical terms and proper nouns are often mistranscribed. Spell them out if needed ("J-A-N-E S-M-I-T-H") or correct them manually before submitting.

Dictation stops unexpectedly

Most systems have a timeout after a pause. Speak continuously or tap the dictation button again to resume. Some systems have a maximum dictation length.
