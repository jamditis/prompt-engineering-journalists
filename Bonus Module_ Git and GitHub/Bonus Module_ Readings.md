# Bonus Module: Git and GitHub for journalists

## Required readings

### 1. The visual foundation: understanding Git's mental model

**"Git Will Finally Make Sense After This"** — LearnThatStack (YouTube)
https://youtu.be/Ala6PHlYjmw

*Watching time: ~15-20 minutes*

This video uses animated visuals to explain how Git works under the hood — not by teaching commands, but by showing what actually happens when you commit, branch, and merge. It builds the mental model you need to give Claude Code the right instructions at the right time.

Pay attention to:
- The three areas where your files live: working directory, staging area, and repository
- What a commit actually is — a snapshot, not a save
- How branches work as pointers, not copies of code
- Why merge conflicts happen and what they mean

### 2. Git for non-developers: thinking in snapshots

**"Git for Humans: An Explanation That Finally Makes Sense"** — Tech Media, Medium
https://medium.com/@tech.media.unicorn/git-for-humans-an-explanation-that-finally-makes-sense-72b245dac63e

*Reading time: 10-15 minutes*

A written companion to the visual approach. Explains Git using plain language and analogies rather than terminal commands. Good for reinforcing the concepts from the video in a format you can skim and reference later.

Pay attention to:
- The analogy between commits and checkpoints in a video game
- Why Git is "distributed" and what that means for your work
- How the staging area acts as a drafting table between your working files and the permanent record

### 3. GitHub basics: the shared workspace

**GitHub quickstart guide** — GitHub Docs (official)
https://docs.github.com/en/get-started/quickstart

*Reading time: 15-20 minutes*

GitHub's own walkthrough of creating repositories, making commits, and opening pull requests. You don't need to follow the terminal steps — Claude Code will handle those. Focus on understanding the concepts: what a repository is, what a pull request does, and how collaboration works on the platform.

Pay attention to:
- The relationship between a local repository (on your computer) and a remote repository (on GitHub)
- What a pull request is and why it exists
- The difference between public and private repositories

---

### 4. Why version control matters in journalism

**"How coding agents fit into the newsroom"** — Nick Hagar, Generative AI in the Newsroom
https://generative-ai-newsroom.com/how-coding-agents-fit-into-the-newsroom-e1cfc1ea5a0b

*Reading time: 10 minutes*

You read this in Module 1 or 2. Revisit the sections on project management and code review. Hagar describes how newsroom developers use Git and pull requests to review each other's work before it goes live. The same workflow applies to journalists building tools with Claude Code — version control isn't just for developers.

Pay attention to:
- How code review through pull requests catches errors before they ship
- Why having a project history matters when someone asks "how did we get this number?"
- The connection between version control and editorial accountability
