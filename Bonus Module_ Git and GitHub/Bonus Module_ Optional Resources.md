# Bonus Module: Git and GitHub for journalists

## Optional resources

These are not required but may be useful depending on your interests and experience level.

### Visual learning

**"Git Will Finally Make Sense After This"** — LearnThatStack (YouTube)
https://youtu.be/Ala6PHlYjmw

The video that inspired this module. Uses animated visuals to explain Git's internal model — commits as snapshots, branches as pointers, and the three areas where files live. If you only watch one thing about Git, watch this.

---

**"Git Explained in 100 Seconds"** — Fireship (YouTube)
https://www.youtube.com/watch?v=hwP7WQkmECE

A rapid-fire overview of Git's core concepts in under two minutes. Good for a quick refresh after you've gone through the module material. Covers init, staging, commits, branching, and merging at speed.

---

**Learn Git Branching** — interactive tutorial
https://learngitbranching.js.org/

An interactive, visual playground where you can see how branches, commits, and merges work in real time. The animated diagrams make abstract concepts concrete. You don't need to memorize the commands shown here — the visual model is what matters.

---

### Deeper reading

**"Learn Git concepts, not commands"** — Nico Riedmann, DEV Community
https://dev.to/unseenwizzard/learn-git-concepts-not-commands-4gjc

Matches the philosophy of this module perfectly: understand what Git is doing and you'll always be able to figure out the right action. Covers the working directory, staging, committing, branching, merging, rebasing, and remoting — all explained conceptually before showing any commands.

---

**"A Visual Guide to Git Internals"** — freeCodeCamp
https://www.freecodecamp.org/news/git-internals-objects-branches-create-repo/

Goes deeper into how Git stores data internally — objects, trees, blobs, and refs. Not necessary for daily use, but satisfying if you're curious about what's happening inside the `.git` folder. Good companion to the visual approaches above.

---

### GitHub-specific

**GitHub Skills** — official interactive courses
https://skills.github.com/

Free, interactive GitHub courses you complete inside actual repositories. Covers pull requests, merge conflicts, GitHub Pages, and GitHub Actions. Each course takes 15-30 minutes. The "Introduction to GitHub" and "Review pull requests" courses are most relevant to this module.

---

**"The Simple Guide to Git"** — Roger Dudler
https://rogerdudler.github.io/git-guide/

A no-nonsense, single-page reference for Git. Covers the basics in about five minutes of reading. Good to bookmark as a cheat sheet for the concepts covered in this module.

---

### Journalism applications

**The Markup's data repositories** — GitHub
https://github.com/the-markup

Real-world examples of investigative journalism published as Git repositories. Each project includes data, methodology, and analysis code — all version-controlled. Browsing these shows what a mature, well-documented journalism project looks like on GitHub.

---

**ProPublica's data store** — GitHub
https://github.com/propublica

Another newsroom that publishes data and tools as open-source repositories. Good examples of README files, documentation, and project structure. Notice how each repository has a clear description and organized files — that's what Git enables.
