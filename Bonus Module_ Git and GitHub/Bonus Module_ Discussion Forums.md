# Bonus Module: Git and GitHub for journalists

## Discussion forum prompts

### Discussion 1: The project you wish you'd tracked

**Prompt:**

Describe a journalism project where you lost work, overwrote something important, or couldn't figure out what changed between two versions of a file. It could be a story draft, a data project, a spreadsheet, or a code script.

What happened? How did you recover (or not)? Looking back, at what points would you have asked Claude Code to make a commit — and what would you have written in the commit message?

Reply to at least one classmate's post with a suggestion about how Git could have helped in their specific situation.

---

### Discussion 2: Version control as editorial accountability

**Prompt:**

One underappreciated aspect of Git is the audit trail. Every commit records who made a change, when, and why (via the commit message). In journalism, accountability for published information is central to credibility.

How might version-controlled projects help your newsroom? Consider scenarios like:
- A reader questions a number in a published data analysis
- An editor wants to know when and why a methodology changed
- A collaborator joins a project midway and needs to understand its history
- A FOIA response tracker needs to show what was processed and when

Pick one scenario and describe how Git's commit history would help. Be specific about what you'd want the history to show.

---

### Discussion 3: Collaboration workflows for small teams

**Prompt:**

Many newsroom teams have two to five people working on shared projects — sometimes on the same files. Before Git, the common approach was emailing files, using shared drives with naming conventions (`analysis_v3_FINAL_johns_edits.xlsx`), or just hoping nobody overwrote anyone's work.

Describe how your team currently handles shared projects. Then propose a GitHub-based workflow using the concepts from this module: separate branches for each person's work, pull requests for review, and merging when the work is approved.

What would need to change in your team's habits for this workflow to work? What's the biggest obstacle?
